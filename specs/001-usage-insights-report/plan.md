# Implementation Plan: AI Usage Insights Report

**Branch**: `001-usage-insights-report` | **Date**: 2026-03-14 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-usage-insights-report/spec.md`

## Summary

Generate a single self-contained HTML insights report from local AI tool chat history across
Claude Code, GitHub Copilot VS Code, and GitHub Copilot CLI. A single Python 3 stdlib script
reads the three data sources, computes usage metrics (patterns, flow state, category, model,
mode), fetches Chart.js once at generation time, and writes a polished visual report to
`~/Desktop/myusage-report.html`. The skill's SKILL.md instructs the agent to run this script.

## Technical Context

**Language/Version**: Python 3.8+
**Primary Dependencies**: None (stdlib only). Chart.js 4.x fetched from CDN at generation time and inlined into HTML output.
**Storage**: Read-only access to local filesystem data sources; writes one HTML file to Desktop.
**Testing**: Python `unittest` (stdlib) with synthetic fixture data
**Target Platform**: macOS (data source paths are macOS-specific in v1; Linux support deferred)
**Project Type**: CLI script invoked by a Claude Code / Copilot skill
**Performance Goals**: Full report generated in under 30 seconds (SC-001)
**Constraints**: Zero pip installs. Single output file. Must run from Claude Code and Copilot CLI.
**Scale/Scope**: Personal use — months of chat history, typically hundreds to low thousands of sessions

## Phase 0: Research

All unknowns resolved. See [research.md](./research.md) for full decisions and rationale.

| Question | Decision |
|---|---|
| Chart.js inlining | Fetch 4.x from jsdelivr CDN at generation time via `urllib.request`, embed inline |
| Categorisation | Keyword scoring — weighted keyword lists per bucket, highest score wins |
| Copilot CLI event chain | Ignore `parentId` — filter by event type and sort by timestamp |
| Claude Code session boundaries | Group by `sessionId` field present on every JSONL entry |
| Timezone normalisation | Parse all timestamps to UTC-aware `datetime` on read |

## Phase 1: Design

### Data Model

See [data-model.md](./data-model.md) for full field definitions. Key entities:

- **Message** — individual user/assistant turn with timestamp, content, char_count
- **Session** — contiguous conversation with derived metrics (duration, flow state, category, mode, model)
- **ToolSnapshot** — aggregated metrics for one tool across all its sessions
- **InsightsReport** — top-level container combining all snapshots

### generate_report.py — Internal Structure

The script has five logical sections, each a plain function (no classes required):

```
1. parse_claude_code(claude_dir)   → List[Session]
2. parse_copilot_vscode(vscode_dir) → List[Session]
3. parse_copilot_cli(cli_dir)      → List[Session]
4. compute_metrics(sessions)       → InsightsReport
5. render_html(report)             → str  (fetches Chart.js, builds HTML)
```

`main()` wires these together, writes the file, and prints the output path.

### HTML Report Sections (in order)

1. **Headline Summary** — top 1 insight per category (personalized digest)
2. **Tool Usage Overview** — session/message split across tools (doughnut + bar chart)
3. **Usage Patterns** — hourly heatmap, day-of-week bar chart
4. **Conversation Depth** — session length distribution, flow state callouts
5. **Chat Categories** — conversation bucket breakdown per tool (stacked bar)
6. **Mode Breakdown** — agent vs ask split per tool (where available)
7. **Model Usage** — model frequency over time per tool

### CLI Interface

See [contracts/cli.md](./contracts/cli.md). Key defaults:
- `--output ~/Desktop/myusage-report.html`
- `--days` (all history if omitted)
- Source dir overrides for each tool (for evals/testing against fixtures)

Updated via `.specify/scripts/bash/update-agent-context.sh claude` after Phase 1.

## Complexity Tracking

| Item | Why Needed | Simpler Alternative Rejected Because |
|---|---|---|
| Chart.js (client-side) | Interactive charts required by spec (FR-003, FR-010) | Pure SVG — far more code to maintain and produces inferior interactivity |
| CDN fetch at generation time | Self-contained report with no runtime dep | Bundling Chart.js in repo — adds ~200KB binary to version control, stales on security updates |
